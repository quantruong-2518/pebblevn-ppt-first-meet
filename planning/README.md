# planning/ — Phễu ý tưởng & thăm dò

Nơi ý tưởng **ra đời và được khám phá**, trước khi cam kết nguồn lực. Đây là chỗ rẻ nhất để sai.

## Quy tắc

- Mọi ý tưởng mới vào đây trước. Đừng nhảy thẳng sang `roadmap/` hay `projects/`.
- Mỗi ý tưởng = 1 file `kebab-case.md` (hoặc `YYYY-MM-DD-ten.md`), tạo từ `_templates/plan.md`.
- Ý tưởng thô, một dòng cũng được — gom vào `backlog.md`. Khi đáng đào sâu thì tách ra file riêng.
- Trạng thái dùng frontmatter: `idea` → `proposed` (đã đủ chín để GM cân nhắc).

## Khi nào thăng cấp lên `roadmap/`?

Khi GM **duyệt** và sẵn sàng phân bổ thời gian/nguồn lực: status đổi sang `approved`,
`chief-of-staff` di chuyển file sang `roadmap/` theo mẫu sáng kiến.

## Dọn dẹp

Hằng tháng: ý tưởng đã chết → đổi status `dropped` hoặc `parked` (kèm 1 dòng lý do), không xóa hẳn để giữ bài học.
