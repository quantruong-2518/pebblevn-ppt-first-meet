# memory/ — Ngân hàng trí nhớ của công ty

> "2nd memory": nơi lưu **những sự thật quan trọng cần nhớ và dùng lại** trong suốt quá trình
> làm công ty. Phiên chat sẽ quên; file ở đây thì không. Mọi agent đọc `INDEX.md` đầu mỗi phiên.

## Lưu cái gì vào đây?

Những thông tin **quan trọng, ổn định, sẽ còn dùng lại**:
- 👤 **Người** — lãnh đạo đối tác, nhà đầu tư, khách hàng, luật sư, đầu mối quan trọng.
- 🤝 **Cam kết / thỏa thuận** — điều đã hứa/đã chốt (kể cả bằng miệng), điều khoản quan trọng.
- 🔢 **Con số** — giá nhập, biên lợi nhuận, chỉ tiêu, mốc thời gian, quy mô thị trường.
- 💡 **Bài học** — điều rút ra, "lần sau nhớ làm/đừng làm".
- 📅 **Sự kiện** — cuộc gặp quan trọng, kết quả & việc cần làm sau đó.
- ⚠️ **Rủi ro** — điều cần cảnh giác, lằn ranh không được vượt.
- 📎 **Tham chiếu** — thuật ngữ, mã số, đường dẫn hay phải tra.

**KHÔNG lưu vào đây:** ý tưởng đang bàn (→ `planning/`), lý do quyết định lớn (→ `company/decisions/`),
hồ sơ nhận diện công ty (→ `company/profile.md`). Trí nhớ là **sự thật nguyên tử**, không phải tài liệu dài.

## Cách ghi (3 cách)

1. **Qua chat (khuyến nghị):** nói *"nhớ giúp tôi: …"* — agent sẽ tạo file + cập nhật `INDEX.md`.
2. **Tự tay:** sao `_template.md` → `ten-ngan-gon.md`, điền frontmatter, thêm 1 dòng vào `INDEX.md`.
3. Mỗi file = **một sự thật**. Tên file `kebab-case`. Cập nhật file cũ thay vì tạo trùng.

## Cách nhớ lại (recall)

- `INDEX.md` là mục lục — lướt nó để biết "ta đang nhớ những gì".
- Khi cần dùng, mở file tương ứng. Agent tự làm việc này khi liên quan tới câu hỏi của bạn.

## Vệ sinh trí nhớ

- Sự thật sai/hết hạn → **sửa hoặc xóa** (và xóa dòng trong `INDEX.md`). Trí nhớ sai còn tệ hơn không nhớ.
- Liên kết các trí nhớ liên quan bằng đường dẫn để tạo mạng lưới ngữ cảnh.
