# projects/ — Thực thi tool/sản phẩm cấp công ty

Nơi sáng kiến trên `roadmap/` được **xây thành deliverable thật**: tool nội bộ, sản phẩm,
website, tài liệu lớn... Mỗi project là **một thư mục con tự chứa**.

## Quy tắc

- Tên thư mục: `projects/<ten-project-kebab-case>/`.
- Khởi tạo từ `_templates/` — bắt buộc có `README.md` với frontmatter trạng thái, mục tiêu,
  phạm vi, và **tiêu chí hoàn thành**.
- Đặc tả (spec) trước khi xây — để GM duyệt phạm vi.
- Link ngược về sáng kiến nguồn trong `roadmap/`.
- Project là code thì cứ chứa code, package.json, v.v. ngay trong thư mục đó — nó tự trị.

## Vòng đời

`planning` (spec, chờ duyệt) → `in-progress` (đang xây) → `done` (đã giao) / `parked`.
