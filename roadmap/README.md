# roadmap/ — Lộ trình (đã cam kết)

Những gì **GM đã duyệt** đưa vào thực hiện. "Đã vào plan". Khác với `planning/` ở chỗ:
ở đây có **ưu tiên, mốc thời gian, và người/agent phụ trách**.

## Quy tắc

- Mỗi mục đến từ `planning/` sau khi được duyệt (status `approved`). Tạo từ `_templates/initiative.md`.
- Bắt buộc có: ưu tiên (P0/P1/P2), mốc thời gian **tuyệt đối** (YYYY-MM-DD), agent phụ trách.
- Giữ số lượng mục "đang làm" nhỏ. Một công ty nhỏ tập trung vài việc còn hơn dàn trải.

## Trạng thái dòng chảy

`approved` (đã duyệt, chờ làm) → `in-progress` (đang làm, có project) → `done` / `parked`.

## Khi nào sinh ra `projects/`?

Khi một sáng kiến cần xây deliverable cụ thể: `product-lead` (hoặc agent phù hợp) tạo thư mục
trong `projects/`, và link hai chiều giữa file roadmap ↔ project.
